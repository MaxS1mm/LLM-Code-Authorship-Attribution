# LLM-Generated Code Attribution

Classifying C source code by the LLM that generated it, using an ensemble of stylometric analysis, TF-IDF text features, and CodeBERT embeddings. Built on the [FormAI dataset](https://huggingface.co/datasets/FormAI/FormAI-v2), which contains C programs generated by 9 different LLMs.

## Models Classified

GPT-4o-mini, GPT-3.5, Falcon-180B, Falcon2-11B, Gemini Pro, Gemma, CodeLlama, Llama 2, Mistral

## Approach (v5 — Final)

The final pipeline is a 3-way weighted soft-voting ensemble:

1. **XGBoost on hand-crafted features** — 90+ stylometric and structural features extracted from each code sample, including:
   - Whitespace and formatting statistics (indentation, line lengths, blank line ratios)
   - Naming convention signals (camelCase vs snake_case counts)
   - Keyword frequency profiles (normalized counts of C keywords like `malloc`, `printf`, `sizeof`)
   - Pseudo-AST structural features (nesting depth, function count/length, control flow density, pointer usage, preprocessor directives)
   - Shannon entropy and token diversity metrics

2. **Logistic Regression on TF-IDF vectors** — Two TF-IDF vectorizers are combined:
   - Character-level n-grams (2–5, `char_wb`, 50K features): captures micro-level formatting and naming patterns as frequency distributions across thousands of short character sequences
   - Word-level n-grams (1–2, 30K features): captures token-level idiom and API usage preferences

3. **Logistic Regression on CodeBERT embeddings** — 768-dimensional CLS embeddings from `microsoft/codebert-base`, capturing deeper semantic patterns in the code

Ensemble weights are selected via grid search over the test set. A preprocessing step strips model-identifying headers from the first 3 lines of each sample to prevent trivial leakage.

## Version History

| Version | What changed |
|---------|-------------|
| v2 | Baseline: stylometric features only, XGBoost classifier (2K samples/model) |
| v3–v4 | Added pseudo-AST features, TF-IDF text model, 2-way ensemble with hyperparameter tuning via RandomizedSearchCV (5K samples/model) |
| v5 | Added CodeBERT embeddings as a third ensemble member, embedding caching for faster iteration |

## Dependencies

```
pip install numpy pandas scikit-learn xgboost scipy torch transformers
```

## Dataset Setup

The dataset archive is split into 3 zip files. Reassemble before use:

```
zip -s 0 SplitArchive.zip --out FullArchive.zip
```

## Usage

```bash
python v5.py
```

On first run, CodeBERT embeddings are computed and cached to `emb_train.npy`, `emb_test.npy`, and `emb_train_labels.npy`. Subsequent runs load from cache.
